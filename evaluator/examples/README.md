# Evaluation Examples

This folder contains real examples of job evaluations using the Pursuit evaluator.

## What's Here

Each example shows:
- The original job description (or summary if too long)
- The evaluator's output (match analysis, stories, gaps, positioning)
- The decision made (Pursue/Maybe/Pass)
- Outcome (if the user applied - did they get an interview? offer? rejection?)

## Why Examples Matter

Seeing real evaluations helps you:
- Understand what "60-70% fit" actually looks like in practice
- Calibrate your own decision-making
- Learn how to position gaps and bridge domain differences
- See the evaluator's reasoning process

## Anonymization

When contributing examples:
- Company names can stay (it's public job postings)
- Anonymize any personal details from your background if needed
- Remove internal/confidential information
- Focus on the evaluation framework, not doxxing yourself

## Contributing Examples

If you use the evaluator and want to share your results:

1. Create a new markdown file: `YYYY-MM-DD-company-role.md`
2. Include:
   - Brief JD summary or key requirements
   - Full evaluator output
   - Your decision (Pursue/Maybe/Pass)
   - Reasoning for any deviation from evaluator recommendation
   - Outcome (if you applied)
3. Submit a PR or open an issue with your example

## Example Structure

```markdown
# [Company Name] - [Role Title]
**Date Evaluated:** YYYY-MM-DD
**Decision:** Pursue | Maybe | Pass
**Outcome:** Applied | Not Applied | Interviewed | Rejected | Offer | Unknown

## Job Description Summary
[Key requirements, responsibilities, must-haves]

## Evaluator Output
[Full output from the prompt]

## User Decision
[Why you chose to Pursue/Maybe/Pass - especially if different from evaluator]

## Outcome Notes
[What happened after - interview? rejection? lessons learned?]
```
