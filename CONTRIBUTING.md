# Contributing to Pursuit

Thank you for considering contributing to Pursuit. This project is built around a philosophy, not just code.

---

## Philosophy First

Before contributing, understand what Pursuit **is** and **isn't**:

### Pursuit IS:
- **Anti-mass-apply.** Quality over volume. Intentional pursuit over desperate spray.
- **Fit-first.** Helping people find jobs they're actually qualified for and excited about.
- **Story-driven.** Your background tells a narrative. Pursuit surfaces it.
- **Realistic.** 60-70% match is strong. You don't need 90%+ to apply.
- **Empowering.** Giving people clarity to pursue with confidence or pass without guilt.

### Pursuit IS NOT:
- One-click mass application tools
- Resume spam generators
- Keyword-stuffing optimizers
- "Apply to 500 jobs with AI" nonsense
- Replacing human judgment with automation

**If your contribution moves Pursuit toward mass-apply, it won't be merged.**

---

## What We Need

Contributions that align with the philosophy:

### 1. Evaluation Examples
- Share real job evaluations (anonymize personal details if needed)
- Document outcomes (Did you apply? Interview? Offer? Rejection?)
- Help others calibrate their decision-making

See [`evaluator/examples/README.md`](evaluator/examples/README.md) for format.

### 2. Prompt Improvements
- Better evaluation frameworks
- Clearer output structure
- Edge case handling (vague JDs, unrealistic requirements, red flags)
- Industry-specific evaluation criteria

Submit a PR with before/after examples showing the improvement.

### 3. New Modules
Want to build Scanner, Strategist, or Tracking?
- Open an issue first to discuss approach
- Align with philosophy (no mass-apply features)
- Start small, iterate, test with real use cases
- Document clearly (READMEs, examples, usage guides)

### 4. Documentation
- Clarify confusing sections
- Add real-world usage examples
- Improve onboarding for new users
- Translate to other languages (if demand exists)

### 5. Integrations
- Tools that complement Pursuit (Notion templates, Airtable bases, scripts)
- Workflows that connect modules (Evaluator → Strategist → Tracking)
- APIs or automation (LinkedIn parsing, company research, referral pathfinding)

**Keep integrations optional.** Not everyone uses Notion or Airtable. Make it modular.

---

## What We Don't Need

Contributions that violate the philosophy:

- ❌ "Apply to all jobs in one click" features
- ❌ Automated mass resume submissions
- ❌ Scrapers that spam recruiters
- ❌ Tools that optimize for volume over fit
- ❌ Keyword stuffing or resume fakery

**We will reject these.**

---

## How to Contribute

### Small Changes (Typos, Docs, Examples)
1. Fork the repo
2. Make your changes
3. Submit a PR with clear description
4. We'll review and merge if aligned

### Larger Changes (New Features, Modules)
1. Open an issue first to discuss
2. Get alignment on approach
3. Build in a branch
4. Submit PR with:
   - What you built and why
   - How it aligns with philosophy
   - Real-world testing/examples
   - Documentation (README, usage guide)

### Bug Reports
If something doesn't work:
1. Describe the issue clearly
2. Include steps to reproduce
3. Share expected vs actual behavior
4. Suggest a fix if you have one

---

## Code of Conduct

Be respectful. Be constructive. Be honest.

- **Respect:** No hostility, no gatekeeping, no elitism.
- **Constructive:** Critique ideas, not people. Suggest improvements, not just complaints.
- **Honest:** If something doesn't work, say so. If you disagree with the philosophy, that's fine — but don't try to change it into mass-apply tooling.
